## 一、现状问题分析（三层负荷视角）

| 负荷类型 | 当前痛点 |
|---------|---------|
| **切换代价** | 多 Session 并行时，每次切换丢失上下文，恢复成本高 |
| **上下文上限** | Session 数量增加后，用户无法同时追踪所有任务状态 |
| **焦虑负荷** | AI 在后台悄悄做了什么？错误谁来兜底？用户心里没底 |

---

## 二、Phase 1：降低切换代价（单任务流优化）

### 目标：减少任务间切换的认知恢复成本

#### 1.1 单任务专注模式 ✅
- **功能**：一次只展示一个 Session 的完整上下文
- **交互**：左/右箭头键切换 Session，切换时显示"加载上下文…"动画
- **效果**：避免多窗口并行导致的 23 分钟恢复成本

#### 1.2 进度连续性记忆 ✅
- **功能**：记录用户上次查看的 Session 位置，下次直接恢复
- **实现**：SQLite 记录 `last_session_id` + `last_task_id`
- **效果**：每次打开都是上次中断的地方，不需要重新定位

#### 1.3 Session 分组管理 ✅
- **功能**：按项目/Sprint 给 Session 打标签，同组内顺序处理
- **交互**：点击标签切换到该组的第一个未完成 Session
- **效果**：同组切换成本低于跨组切换

## 三、Phase 2：突破上下文上限（AI 辅助上下文管理）

### 目标：不让用户同时追踪超过 7±2 个对象

#### 2.1 AI 上下文摘要（认知负荷 ↓）✅
- **功能**：每个 Session 自动生成 3 句话摘要
  - 当前进度是什么？
  - 遇到什么阻碍？
  - 下一步建议做什么？
- **交互**：Session 卡片上直接显示摘要，不需要点进去看
- **效果**：工作记忆从"理解整个 Session"降到"看 3 句话"

#### 2.2 智能任务聚合
- **功能**：相似/相关任务自动聚合为一组，显示为一个"任务簇"
- **算法**：基于标题语义相似度 + 同一 Session 内的任务关系
- **效果**：列表从 20 项任务变成 5 个任务簇，上下文压力骤降

#### 2.3 状态主动推送 ✅
- **功能**：Session 状态重大变化时，主动推送通知给用户
  - ✅ AI 完成了某个阶段
  - ⚠️ AI 遇到了不确定的问题
  - 🔴 AI 疑似出错，需要人工确认
- **交互**：通知面板显示，点击直接跳转到对应 Session
- **效果**：用户不需要主动监控，重要变化不会漏掉

## 四、Phase 3：消除焦虑负荷（信任体系建设）

### 目标：让用户相信"AI 干的事我在掌控中"

#### 3.1 变更日志（Change Log）✅
- **功能**：AI 每次操作都记录：
  - 时间戳
  - 操作类型（读取/修改/创建/删除）
  - 操作前状态
  - 操作后状态
  - 置信度评分
- **交互**：每个 Session 有一个"历史"面板，可以回滚到任意版本
- **效果**：AI 做了什么全部透明，焦虑来源从"不知道"变成"有据可查"

#### 3.2 置信度指标系统 ✅
- **功能**：每个 AI 输出显示可信度评分（0-100%）
  - 🟢 95%+：高置信，AI 自动执行
  - 🟡 70-95%：中置信，通知用户
  - 🔴 <70%：低置信，暂停等待人工确认
- **算法**：基于历史准确率 + 任务复杂度 + 上下文完整性
- **效果**：用户知道什么时候该信任 AI，什么时候该介入

#### 3.3 异常自动拦截 ✅
- **功能**：AI 输出触发以下条件时自动暂停：
  - 触及敏感操作（删除文件、提交代码）
  - 输出内容包含"不确定"、"可能"、"建议人工确认"
  - 连续 3 次生成结果相似度 > 90%（可能陷入循环）
- **交互**：弹窗显示"AI 遇到以下情况需要您确认"，用户一键选择"确认/修改/取消"
- **效果**：把焦虑来源从"AI 可能犯错"变成"AI 犯错前我就拦住了"

#### 3.4 AI 行为可视化面板 ✅
- **功能**：Dashboard 增加一个"AI Activity"面板
  - 实时显示当前有哪些 AI 在运行
  - 每个 AI 的当前状态（思考中/生成中/等待中/已完成）
  - 异常标记（长时间无响应/错误次数过多）
- **效果**：所有 AI 状态一目了然，不用担心"哪个 AI 在偷偷摸鱼"

## 五、Phase 4：协作模式重构（长期目标）

### 目标：把 AI 定位从"协作伙伴"回归到"高效工具"

#### 4.1 标准操作流程（SOP）化 ✅
- **功能**：用户可以为每类任务定义 SOP
  - 触发条件（什么任务走这个 SOP）
  - 执行步骤（AI 依次做什么）
  - 输出标准（结果长什么样才算合格）
  - 验收规则（怎么判断完成了）
- **效果**：减少临时判断，把 AI 从"协作伙伴"变成"听话的执行者"

#### 4.2 人类决策边界明确化 ✅
- **功能**：设置"人类保留决策"清单
  - 涉及代码提交的变更 → 必须人工确认
  - 涉及多 Session 协调 → 必须人工确认
  - 涉及外部 API 调用 → 必须人工确认
  - 其他 → AI 可自动执行
- **效果**：人类只做高价值的决策，低价值执行全部交给 AI

## 六、新增功能 (v2.3.0)

### 6.1 多视图切换系统 ✅

| 视图 | 快捷键 | 适用场景 |
|------|--------|---------|
| **卡片视图** | 点击▦ | Session数>10，高信息密度一目了然 |
| **列表视图** | 点击☰ | 深度展开，显示所有详情 |
| **紧凑视图** | 点击▤ | Session数>20，快速浏览 |

**交互**：
- 右上角视图切换按钮
- 切换后自动记忆用户偏好

### 6.2 Session 健康度系统 ✅

**健康度计算**：
```
health = 50 (基础分)
       + 进度加分 (完成率 × 0.2)
       + 无阻塞加分 (无阻塞? 20 : max(0, 20 - 阻塞数×5))
       + 活跃度加分 (30分钟内活动? 10 : 0)
       - 超时惩罚 (持续>2h? 10 : 0)
```

**显示指标**：
- 🏥 健康度百分比（0-100）
- 📋 任务进度（已完成/总数）
- ⚡ Token速率（tokens/分钟）
- ⚠️ 阻塞警告

**优先级标签**：
- 🔴 紧急 (critical)
- 🟠 高 (high)
- 🔵 中 (medium)
- 🟢 低 (low)

### 6.3 自然语言任务解析 ✅

**支持格式**：
| 输入 | 解析结果 |
|------|---------|
| `创建用户登录功能` | 任务：创建用户登录功能，标签：feature |
| `修复支付bug` | 任务：修复支付bug，优先级：高，标签：bug,fix |
| `紧急：数据库迁移` | 任务：数据库迁移，优先级：紧急 |
| `完成文档编写` | 任务：完成文档编写 |

**API**：
```
POST /api/tasks/parse
Body: { text: "创建用户登录功能", sessionId?: "xxx", projectKey?: "xxx" }
Response: { task: {...}, parsed: { title, priority, tags, type } }
```

### 6.4 AI 3句话摘要 ✅

**摘要格式**：
```
当前会话状态（3句总结）：
1. 正在进行「用户模块重构」，完成了60%功能点
2. 遇到 API 兼容性问题阻塞，预计影响2个任务
3. Token 消耗速率正常，本会话已消耗 15K tokens
```

**自动生成时机**：
- Session 创建时
- 每 5 分钟刷新
- 用户查看详情时
- 任务状态变更时

**API**：
```
GET /api/sessions/summary/:sessionId
Response: { summary: "正在...。遇到...。建议...", health: {...} }
```

## 八、API 完整列表

### Session APIs
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/sessions` | 获取所有会话 |
| GET | `/api/sessions/views?mode=health` | 按健康度排序 |
| GET | `/api/sessions/views?mode=priority` | 按优先级排序 |
| GET | `/api/sessions/views?mode=recent` | 按最近活动时间排序 |

| GET | `/api/sessions/summary` | 获取所有会话AI摘要 |
| GET | `/api/sessions/summary/:sessionId` | 获取单个会话AI摘要 |

| POST | `/api/tasks/extract-from-message` | 从消息中提取任务 |
